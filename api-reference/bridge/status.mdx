---
title: "Get Bridge Status"
api: "GET /bridge/status/:sessionId"
description: "Check the status of a bridge request"
---

## Path Parameters

<ParamField path="sessionId" type="string" required>
  The unique session ID returned when creating a bridge request
</ParamField>

## Response

<ResponseField name="sessionId" type="string">
  The unique identifier for this bridge session
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the bridge request. One of:
  - `pending_deposit` - Waiting for USDT deposit
  - `processing` - USDT received, executing bridge
  - `completed` - Bridge successful
  - `failed` - Bridge failed
</ResponseField>

<ResponseField name="sourceChain" type="string">
  Source blockchain network
</ResponseField>

<ResponseField name="destinationChain" type="string">
  Destination blockchain network
</ResponseField>

<ResponseField name="amountUSDT" type="string">
  Amount of USDT being bridged
</ResponseField>

<ResponseField name="destinationWallet" type="string">
  Wallet address receiving USDT on destination chain
</ResponseField>

<ResponseField name="depositAddress" type="string">
  Smart wallet address where USDT should be deposited
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp when the bridge request was created
</ResponseField>

<ResponseField name="txHash" type="string">
  Transaction hash (only present when status is `completed`)
</ResponseField>

<ResponseField name="explorerUrl" type="string">
  Block explorer link to view the transaction (only when completed)
</ResponseField>

<ResponseField name="layerZeroUrl" type="string">
  LayerZero scanner link to track cross-chain message (only when completed)
</ResponseField>

<ResponseField name="error" type="string">
  Error message (only present when status is `failed`)
</ResponseField>

## Status Flow

<Steps>
  <Step title="pending_deposit">
    Waiting for USDT to be sent to the deposit address
  </Step>
  <Step title="processing">
    USDT received, getting quote and executing bridge transaction
  </Step>
  <Step title="completed">
    Bridge successful, USDT delivered to destination
  </Step>
</Steps>

## Polling for Updates

You can poll this endpoint to monitor progress:

```javascript
async function waitForBridgeCompletion(sessionId) {
  const maxAttempts = 120; // 20 minutes (10s intervals)
  
  for (let i = 0; i < maxAttempts; i++) {
    const response = await fetch(`https://your-api.com/bridge/status/${sessionId}`);
    const data = await response.json();
    
    console.log(`Status: ${data.status}`);
    
    if (data.status === 'completed') {
      console.log('✅ Bridge completed!');
      console.log('TX Hash:', data.txHash);
      return data;
    }
    
    if (data.status === 'failed') {
      console.error('❌ Bridge failed:', data.error);
      throw new Error(data.error);
    }
    
    await new Promise(resolve => setTimeout(resolve, 10000)); // Wait 10 seconds
  }
  
  throw new Error('Timeout waiting for bridge completion');
}
```

<Info>
  Bridge completion typically takes 2-5 minutes depending on network congestion.
</Info>

## Error Handling

<ResponseExample>

```json Session Not Found
{
  "error": "Session not found"
}
```

</ResponseExample>

If you receive a "Session not found" error, verify:
- The session ID is correct
- The session hasn't expired (sessions are stored in-memory)
